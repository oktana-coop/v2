# Paged.js PDF Engine

Exporting to PDF using [Paged.js](https://pagedjs.org/). There are two implementations: an Electron adapter that coordinates across processes for programmatic PDF generation, and a browser adapter that uses the browser's native print dialog.

## How it works in Electron

PDF export spans three Electron processes, coordinated via IPC:

```
Renderer (main window)        Main Process        Renderer (hidden print window)
       |                          |                           |
       |-- IPC: print-to-pdf ---->|                           |
       |                          |-- create BrowserWindow -->|
       |                          |-- load print.html ------->|
       |                          |-- send HTML + wait ------>|
       |                          |                           |-- inject stylesheet
       |                          |                           |-- Paged.js pagination
       |                          |                           |
       |                          |                           |-- dispatch 'pagedjs:rendered'
       |                          |<- ready ------------------|
       |                          |                           |
       |                          |-- print to PDF ---------->|
       |                          |<- PDF buffer -------------|
       |                          |-- close window            |
       |<- PDF bytes -------------|                           |
```

1. The **main window renderer** sends the HTML to the main process via the `print-to-pdf` IPC channel.

2. The **main process** creates a hidden BrowserWindow, loads `print.html`, sends the HTML content, and waits for the `pagedjs:rendered` event.

3. The **hidden print window** receives the HTML, injects the default stylesheet, and runs Paged.js to paginate the content. Paged.js reads the `@page` rules from the stylesheet and splits the content into pages. Once done, it dispatches `pagedjs:rendered`.

4. The **main process** generates a PDF from the paginated content with CSS-controlled page size and zero margins, so Paged.js fully controls the layout. The resulting PDF buffer is returned to the main window renderer.

### Why a hidden BrowserWindow?

Paged.js is a document renderer that expects full control of the page and it needs a real DOM to paginate content. It measures element heights, splits content across pages, and generates CSS regions. This requires a full browser rendering context, which only a BrowserWindow provides in Electron. A separate lightweight entry point (`print.html`) is used to avoid loading the full application bundle.

## How it works in the browser

The browser adapter creates a hidden iframe, runs Paged.js pagination inside it, and triggers the browser's native print dialog. The user can then save the result as a PDF through the browser's "Save as PDF" print destination.

Since the browser doesn't provide programmatic access to PDF bytes, the adapter returns an empty buffer. The actual PDF is generated by the browser's print subsystem and saved by the user.
